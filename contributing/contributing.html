Contributing

Bonfhir is open to all contributions in order to enrich the ecosystem!

You can contribute in different ways:

  • enrich / fix this documentation
  • submit and fix bugs
  • submit feature requests and new ideas
  • submit brand new packages to add new features

Please do not hesitate to reach out and discuss your contributions before starting any work!

Please make sure that your contributions align with the project orientations.

Running locally​

All Bonfhir packages are running in a monorepo. This is how you can setup your local development environment.

Pre-requisites​

The solution is configured to run in a Development Container.

  • Docker up and running
    • See this page for more precise requirements on Docker
  • Visual Studio Code installed
  • VS Code Dev Containers extension is installed

Open the solution​

  • Clone the repo: https://github.com/bonfhir/bonfhir
  • Open in Visual Studio Code and run the Dev Containers: Open Folder in Container... command

Verify that this is working properly by running the following commands:

node ➜ /workspace (main ✔) $ yarn build

node ➜ /workspace (main ✔) $ yarn checks

node ➜ /workspace (main ✔) $ yarn test

Organization​

The monorepo uses lerna and yarn workspaces.

Individual packages can be found in the packages directory.

Each package should expose a list of standard scripts:

  • yarn build: Create a production build of the code (compiled with TypeScript)
  • yarn check: Run quality checks: Prettier, ESLint and type checking
  • yarn clean: Clean artifacts related to the build process; may be invoked automatically by the build script
  • yarn codegen: Run the code generation on the package source files, to generate source code
  • yarn test: Run automated tests
